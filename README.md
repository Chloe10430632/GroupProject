【團隊開發 Git 規範】

• 分支管理
- 禁止直接推送到 origin/main：所有功能開發請切換到自己，feature/ 你的分支。
- 命名規範：feature/功能名稱 (例如：feature/login-api)，可以參考commit message網頁命名規則。
- 更新頻率："每天19:00" 完成部分進度就提交，合併完成後再繼續寫，避免最後產生大規模衝突。

• 開發流程
1. 更新本地main分支：開發前執行提取遠端origin/main分支，確保自己的檔案在最新狀態再開始寫 !!
2. 切換分支：VScode左下角的分支，點選你的分支 feature/your-name(你的英文名)。
3.合併origin/main分支到自己的分支：每天合併過後後，記得merge合併到自己的分支，保持檔案維持在最新狀態。
4. 提交紀錄："feat: 描述功能" (簡潔明瞭為主)。
5. 發起 PR：推送至 GitHub 後發起 Pull Request，等待組長/副組長審核。

• Notion 同步
- 領取任務時，將 Notion 狀態改為 Doing。
- 發起 PR 時，狀態改為 In Review。
- 合併完成後，狀態改為 Done。

✹ 若發生合併衝突處理 ✹ 
- 若你的檔案發生 Conflict，組長告知合併衝突組員，請組員先將 main 合併回自己的分支進行排解，確定沒問題後再推送。
- 無法自行處理時，請務必告知組長，切勿強行覆蓋(雖然有做不能直接git push到origin main的設定，但如果還是成功強行覆蓋main分支......你就準備請各位組員喝飲料唄XD)。
